# PromoFlow 系统设计文档

1. 项目概述

PromoFlow 是一个基于飞书工作台的营销内容管理系统，旨在为有方大健康的各类员工（如招商、投放、普通员工等）提供一个集中的平台来管理、上传和审核营销素材。系统将利用 AI 技术自动化地分析内容并生成摘要，同时集成 AI 机器人提供智能内容通知和检索功能。

主要功能
	•	内容浏览与筛选：用户可以浏览最新内容，按类别、标签等筛选素材。
	•	内容上传与审核：普通员工可以上传图片、视频等素材，并填写标题、标签；审核人员负责内容的审核和分类管理。
	•	AI 内容分析与生成：AI 自动分析上传的内容，生成摘要、关键词，支持内容的智能检索。
	•	AI 机器人：通过飞书机器人，自动在群聊中通知相关人员内容的审核通过，同时支持自然语言查询相关内容。

目标用户
	•	普通员工：上传素材，选择标签和标题等信息。
	•	审核人员：审核上传内容，修改标签、分类、描述等信息。
	•	管理员：管理系统设置、调整类别、标签等。

2. 系统架构设计

2.1 架构概览

系统将由前端、后端、AI 模型、数据库等多个部分组成，整体架构如下：
	•	前端：React + Tailwind，用于实现响应式的用户界面，提供内容管理和搜索功能。
	•	后端：Python FastAPI，提供 RESTful API 支持，处理用户请求、内容存储、AI 分析等。
	•	数据库：
	•	PostgreSQL 用于存储元数据（如标题、标签、描述等）。
	•	Milvus 用于存储内容的向量化结果，支持高效的语义检索。
	•	AI 模型：利用 千问多模态模型 和 OpenAI GPT 进行内容分析和生成。
	•	AI 机器人：使用 OpenClaw 作为飞书机器人框架，处理通知和智能查询功能。

2.2 系统组件
	1.	前端界面（React + Tailwind）
	•	提供一个响应式、用户友好的界面，员工可上传和查看素材，审核人员可进行审核，管理员可调整设置。
	2.	后端 API（FastAPI）
	•	处理上传素材、检索素材、生成响应、AI 内容分析等。
	3.	数据库（PostgreSQL）
	•	存储内容的元数据（如标题、标签、描述等），便于查询和管理。
	4.	向量数据库（Milvus）
	•	存储 AI 分析后的向量，支持语义检索，提供相关内容的高效检索。
	5.	AI 内容分析（千问多模态模型 + OpenAI GPT）
	•	对图片、视频、文本进行分析，生成摘要、关键词等信息，并存储为向量供检索使用。
	6.	飞书机器人（OpenClaw）
	•	自动推送内容审核通过通知，支持群内自然语言查询素材。

3. 功能模块设计

3.1 用户管理与权限
	•	普通员工：能够上传素材，填写标题、标签、描述等，查看自己的上传记录。
	•	审核人员：查看待审核的内容，审核并修改内容的标签、分类、描述等。
	•	管理员：管理系统设置，调整类别和标签，查看系统活动记录等。

3.2 内容管理模块
	•	上传功能：员工上传图片、视频等素材，并填写标题、标签等信息，AI 自动对内容进行分析。
	•	审核功能：审核人员可以查看待审核内容，并进行修改或删除，确保内容符合规范。
	•	内容分类与标签管理：管理员可以调整内容分类、标签等信息，以便更好地组织和筛选内容。

3.3 AI 分析与生成模块
	•	内容分析：使用千问多模态模型分析上传的图片、视频或文本，生成内容摘要和关键词，并将其转化为向量存储到 Milvus 中。
	•	智能生成：当用户查询素材时，系统会通过生成模型（如 GPT）生成自然语言响应，基于检索到的相关内容。

3.4 AI 机器人模块
	•	自动通知：当素材通过审核时，飞书机器人会自动在群聊中发送通知，告知大家新内容已通过审核。
	•	自然语言查询：用户可以通过 @ 机器人来查询感兴趣的内容，机器人支持自然语言查询并返回相关素材。

4. 数据库设计

4.1 PostgreSQL

PostgreSQL 用于存储内容的元数据，包括素材的标题、标签、描述、上传者等。数据库的设计如下：

表结构：
	1.	content
	•	id: 主键，自动增长
	•	title: 标题
	•	description: 描述
	•	tags: 标签（JSONB 格式）
	•	status: 状态（待审核、已通过、已拒绝）
	•	created_at: 创建时间
	•	updated_at: 更新时间
	•	uploaded_by: 上传人
	2.	audit_log
	•	id: 主键，自动增长
	•	content_id: 素材ID，外键关联到 content
	•	audit_status: 审核状态
	•	audit_comments: 审核人员的评论
	•	audit_time: 审核时间

4.2 Milvus（向量数据库）

Milvus 用于存储内容的向量化结果，每个上传的内容将被转化为向量并存储。每个向量记录包含：
	•	vector_id: 唯一标识符
	•	content_id: 外键，关联到 PostgreSQL 中的内容表
	•	vector: 向量表示内容的特征（如图像描述、视频摘要等）

4.3 索引设计

在 Milvus 中，我们将根据向量的内容选择合适的索引类型（如 IVF 或 HNSW）来提高检索效率。

5. 技术栈

前端：
	•	React：构建用户界面，支持动态渲染和响应式设计。
	•	Tailwind CSS：快速构建美观、简洁的 UI。
	•	ShadCN：组件库，用于实现常见的 UI 组件。

后端：
	•	FastAPI：用于构建高效的 RESTful API，支持异步处理和自动化文档生成。
	•	PostgreSQL：关系型数据库，存储内容元数据和审核记录。
	•	Milvus：向量数据库，存储分析后的向量数据，支持高效的向量检索。

AI 模型：
	•	千问多模态模型：用于内容分析，将图片、视频等转化为向量并生成摘要。
	•	OpenAI GPT：用于生成自然语言响应，提升内容检索和生成的能力。

AI 机器人：
	•	OpenClaw：用于在飞书中创建并管理机器人，实现自动通知和自然语言查询。

6. 开发与部署

开发环境：
	•	本地开发：使用 Docker 部署 Milvus 和 PostgreSQL，本地开发与测试。
	•	开发工具：VSCode、Git、Docker、Python 环境（FastAPI）等。

部署方案：
	•	开发阶段：可采用单机部署，通过 Docker Compose 快速启动所有服务。
	•	生产阶段：使用分布式部署，确保系统可扩展和高可用性。Milvus 和 PostgreSQL 可以部署在不同的节点上，利用负载均衡和分片来处理大规模数据。

CI/CD：
	•	使用 GitHub Actions 或 Jenkins 来自动化测试、构建、部署流程。

⸻

7. 项目初始化

根据以上设计，AI Coding Agent 可以通过以下步骤初始化项目：
	1.	创建后端项目：使用 FastAPI 初始化后端服务，配置数据库连接（PostgreSQL 和 Milvus）。
	2.	创建前端项目：使用 React 和 Tailwind 初始化前端项目，设计页面和交互。
	3.	AI 集成：集成千问多模态模型进行内容分析，集成 OpenAI GPT 用于生成内容。
	4.	机器人集成：集成 OpenClaw 创建飞书机器人，并配置自动通知和自然语言查询功能。
	5.	数据库设计与迁移：创建 PostgreSQL 数据库表，并使用 Alembic 进行数据库迁移。
